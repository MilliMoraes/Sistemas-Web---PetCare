# **Sistemas Web - PetCare**

## *Milliany Rodrigues de Morais*

## *Título Proposto: Plataforma Digital de Suporte e Bem-Estar Animal: Pet Care Info – Um Guia Essencial para Tutores*

### RESUMO

O projeto propõe o desenvolvimento de uma Plataforma Web Responsiva que atua como um recurso informativo para tutores de animais de estimação. O foco é na Saúde Animal e na facilidade de acesso à informação. A plataforma se estrutura em dois pilares: um repositório de conhecimento com dicas de cuidados e prevenção, e um módulo de geolocalização para identificação rápida de hospitais e clínicas veterinárias próximas. O objetivo é fornecer suporte em situações rotineiras ou de emergência, promovendo o bem-estar animal através da educação e da conectividade com serviços essenciais.

### 1. TEMA

O trabalho final tem como tema o desenvolvimento de uma aplicação web de utilidade pública (Pet Care), concentrada em informações de saúde categorizadas por espécie e localização simplificada de serviços veterinários de emergência e rotina.

### 2. ESCOPO

Este projeto terá as seguintes funcionalidades detalhadas:

2.1 Módulo de Conteúdo e Educação (Knowledge Base)
Página Inicial: Layout intuitivo, responsivo, com destaque para a função de busca.

Organização de Conteúdo: O conteúdo será estritamente categorizado por tópicos (Alimentação, Primeiros Socorros...).

Funcionalidade de Busca: Campo de busca com sugestão de preenchimento que filtra o repositório de artigos por palavra-chave.

Acessibilidade: Garantir que o design e o código sigam as diretrizes básicas de acessibilidade para leitura clara em diferentes dispositivos.

2.2 Módulo de Serviços Locais (Diretório)
Geolocalização e Busca: Permite ao usuário buscar serviços (Clínicas, Hospitais) utilizando a localização atual ou por inserção manual de CEP/Cidade.

Visualização Simples: Os resultados serão apresentados em formato de lista e, opcionalmente, com uma visualização de mapa.

Filtros de Serviço: Possibilidade de filtrar os resultados por tipo de serviço (Ex: 24h, Especialidades).

Detalhe do Serviço (Mockup): Cada item da lista exibirá nome, endereço, contato e horário de funcionamento.

### 3. RESTRIÇÕES

Para manter o foco técnico e a viabilidade do projeto, as seguintes funcionalidades não serão desenvolvidas:

Autenticação e Permissão: A plataforma será totalmente pública. Não haverá gerenciamento de usuários, áreas restritas ou painel de administração.

Interações Dinâmicas Complexas: Não serão implementados sistemas de comentários, formulários de envio de dados (exceto busca simples), ou fóruns de discussão.

Integração de Agendamento: Não haverá qualquer ferramenta para agendar consultas, banhos ou serviços diretamente pela plataforma.

Integração de Pagamento: Não haverá módulos de venda de produtos ou serviços.

Persistência de Dados Crítica: O banco de dados será simples, focado em armazenar e servir o conteúdo. Não haverá armazenamento de dados pessoais de usuários.
